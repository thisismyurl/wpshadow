# This Is My URL Shadow Support

This page explains the best place to go for help, bug reports, feature ideas, and security concerns.

Our goal is to keep support clear, respectful, and genuinely useful.

---

## Quick Routing Guide

### I need help using the plugin
Use:
- the WordPress.org support forum for general user help, or
- the project documentation in [`docs/`](docs/)

### I found a bug
Use:
- GitHub Issues for reproducible bugs and development-tracked problems

### I have an idea for a new feature or improvement
Use:
- GitHub Issues or the feature request template, when available

### I found a security problem
Use:
- [`SECURITY.md`](SECURITY.md) and report it privately

### I found an accessibility barrier or disability-related usability issue
Use:
- GitHub Issues for reproducible accessibility bugs, or
- this support route if you are not sure how to describe the problem technically

### I want to support the project
Use:
- [`DONATE.md`](DONATE.md)

---

## Before Asking for Help

Please include as much of the following as you can:

- This Is My URL Shadow version
- WordPress version
- PHP version
- whether this is single-site or multisite
- what you expected to happen
- what actually happened
- screenshots, logs, or steps to reproduce

This helps us help you faster.

---

## What Support Covers

We try to help with:

- setup and onboarding questions
- understanding findings and diagnostics
- unexpected behavior or regressions
- accessibility or usability problems
- documentation gaps or confusing wording
- disability-related barriers that make the plugin harder to use confidently

Accessibility concerns are welcome here and should not be treated as edge cases.

---

## What Makes a Good Bug Report

A good bug report is:

- specific
- reproducible
- calm and factual
- focused on the user impact

A great bug report often includes:

1. the exact page or action
2. the steps taken
3. the visible error or unexpected result
4. any relevant PHP or browser console output

---

## Our Support Philosophy

This Is My URL Shadow is built around **advice, not sales**.

That means support should never feel like:

- a funnel
- a pressure tactic
- a paywall trap
- a punishment for asking beginner questions

We aim to be:

- respectful
- plain-English
- practical
- accessibility-aware
- honest about what exists today and what is still planned
