# WPShadow Business Model

**Date:** Current
**Status:** ✅ Active — WPShadow Core only

---

## What Exists Today

### WPShadow Core — Free WordPress Plugin

WPShadow Core is the only currently available product. It is:

- **100% free** — no trial, no paywall, no paywalled features
- **Self-contained** — runs entirely on your WordPress server
- **No external services required** — no cloud, no tokens, no registration
- **Open development** — public repository

**What it does:**
- Runs 378 diagnostic tests across 10 categories (performance, security, SEO, accessibility, and more)
- Displays findings in a real-time admin dashboard
- Provides an extensible treatment framework for auto-fix capabilities
- Logs activity and tracks KPI impact
- Integrates with WordPress Site Health

---

## Core Philosophy

### Why It's Free

If it doesn't require ongoing server costs or external services, it's free. Period.

The diagnostic tests run on your WordPress server. There is no cost to us when you run them. Therefore there is no cost to you.

### What We Never Charge For

- ❌ Basic plugin features
- ❌ Access to diagnostic results
- ❌ Knowledge about your site
- ❌ Community support
- ❌ Basic documentation

### What We Would Only Charge For (If Ever)

- ✅ Real server costs (cloud compute, storage)
- ✅ Third-party API services (external scanning, threat data)
- ✅ Priority support with response SLAs
- ✅ Advanced features requiring ongoing infrastructure

## The 5 CANON Pillars

All decisions are evaluated against these pillars:

| Pillar | Principle |
|--------|-----------|
| 🌍 **Accessibility First** | No feature is complete until it's accessible |
| 🎓 **Learning Inclusive** | Support all learning styles (text, visual, hands-on) |
| 🌐 **Culturally Respectful** | Design for global communities |
| 🛡️ **Safe by Default** | Protect users from mistakes and attacks |
| ⚙️ **Murphy's Law** | Defensive engineering with resilient fallbacks |

## Sustainability

WPShadow Core is developed and maintained as a commitment to the WordPress community. The current model is:

- Plugin released free
- Development funded by the project director

Future sustainability options (not yet implemented) may include optional cloud diagnostic services, backup storage, or premium support — all of which would be clearly priced, transparently explained, and never required to use the core plugin.

---

## Transparency Commitment

If paid products are ever introduced, every paid offering must clearly state:

1. **What it costs** — exact pricing
2. **Why it costs** — breakdown of actual infrastructure/service costs
3. **What you get** — features, limits, what's included
4. **What you don't pay for** — the free tier always exists
5. **How we spend it** — revenue allocation breakdown
