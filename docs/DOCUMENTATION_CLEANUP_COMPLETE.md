## Documentation Standards Going Forward

**Only add docs that are:**
- ✅ Foundational (architecture, philosophy, standards)
- ✅ Evergreen (not tied to specific phases or tasks)
- ✅ Maintained (kept current with code/philosophy)
- ✅ Published (meant for team/community to read)

**Don't add docs that are:**
- ❌ Task tracking (use commit messages and issues)
- ❌ One-time reports (preserve in git, not in active docs)
- ❌ Obsolete (delete when no longer relevant)
- ❌ Event-specific (archive when event ends)
