   - Multi-site dashboard (free for 3 sites)

   - Transparent, simple, predictable pricing

   - Annual discount available (no "enterprise contact us")
   - Can downgrade anytime, no penalty

## Feature Decision Framework (Philosophy Checklist)

### ✅ Green Light - Build It

Feature passes when it:
- Makes WordPress management easier for everyone (not just power users)
- Can be done exceptionally well in free tier (local or generous cloud)
- Creates educational opportunity (links to KB/training)
- Delivers measurable KPI improvement (time saved, issues fixed, confidence gained)
- Builds confidence/trust (transparency, intuitive UI, predictable behavior)
- Talk-worthy (users would share: "Check this out!")

**All 6 GitHub issues pass this test.**

### ⚠️ Yellow Light - Proceed with Caution

Feature requires careful design when it:
- Requires infrastructure (must have generous free tier first)
- Complex UI (must be exceptionally intuitive, not just "powerful")
- Collects data (must be consent-first, transparent, clear user benefit)
- Could feel salesy (rework all copy with "helpful neighbor" lens)
- Creates dependency (ensure user learns, not just relies)

**Issue #566 (consent) yellow-lit due to data collection, but philosophy alignment makes it green.**

### 🛑 Red Light - Don't Build

Feature rejected when it:
- Creates dependency instead of empowerment ("only we can do this")
- Requires payment for basic functionality (local features must be free)
- Hides information behind upsells ("upgrade to see this")
- Feels manipulative or sales-driven (pressure, dark patterns, artificial urgency)
- Collects data without clear user benefit (data for our benefit, not theirs)
- "Enterprise only" or "Contact sales" features (transparent pricing only)

**None of the GitHub issues are red-lighted. Users are asking for helpful features.**
