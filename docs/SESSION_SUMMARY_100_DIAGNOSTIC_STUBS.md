## Technical Architecture Notes

### Base Classes Used
- All diagnostics extend pattern: `public static function check(): ?array`
- Namespace: `WPShadow\Diagnostics`
- Return format: `['id' => '', 'title' => '', 'description' => '', 'severity' => '', 'kb_link' => '', 'training_link' => '']`

### Auto-Fixable Diagnostics (Treatments)
- 3 of 100 marked auto-fixable in Part 1
- Requires paired Treatment class extending `Treatment_Base`
- Implements `apply()` and `undo()` methods
- KPI tracking on successful application

### Registry Integration
- All classes auto-discovered via `Diagnostic_Registry`
- PSR-4 autoloading
- No manual registration required
- Categories: database, frontend, caching, security-performance, wordpress, assets, monitoring
