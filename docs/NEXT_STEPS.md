# WPShadow Next Steps

**Last Updated:** April 5, 2026  
**Scope:** Public-facing priorities for the next phase of WPShadow Core

---

## Why This Document Exists

People should not have to guess:

- what WPShadow already does well
- what is still being refined
- what is planned next
- and what is only an idea for later

This page exists to communicate that clearly and honestly.

---

## What Exists Today

WPShadow Core today is a free WordPress plugin focused on:

- diagnostics
- findings visibility
- safe remediation patterns
- admin clarity
- accessibility-first guidance
- trustworthy, plain-English explanations

The current live release exposes **230 display-ready diagnostics** in `Settings → Diagnostics`, along with **101 executable treatment classes** and the current backup, review, and recovery safety workflows.

---

## Current Priorities

### 1. Stabilize and Polish the Core Experience
We want the plugin to feel dependable, consistent, and understandable.

Focus areas include:
- admin page consistency
- clearer finding explanations
- safer defaults and better recovery paths
- fewer surprises in diagnostics and treatment behavior

### 2. Improve Documentation and Trust Signals
We are continuing to make the project easier to understand for both users and reviewers.

That includes:
- clearer source-of-truth documentation
- better WordPress.org messaging
- stronger GitHub community files
- plain-English support and privacy guidance

### 3. Strengthen Accessibility and Inclusivity
Accessibility is not a side quest. It remains a core requirement.

We will continue improving:
- keyboard support
- screen-reader clarity
- readable writing
- less ambiguous UI states
- better explanations for non-technical users
- disability-informed usability review and lower-cognitive-load workflows
- clearer recovery paths for users who are stressed, overwhelmed, or using assistive technology

### 4. Expand Useful, Safe Coverage
New work should favor:
- diagnostics with clear real-world value
- safe, reversible fixes
- better categorization and prioritization
- stronger site-owner confidence

---

## What We Are Not Trying to Be

For now, WPShadow Core is **not** trying to become:

- a bloated all-in-one marketing suite
- a fear-based upsell engine
- a cloud-only dependency
- a plugin that hides basic understanding behind a paywall

If optional services grow later, they should still respect the same core philosophy.

---

## How We Decide What Comes Next

We prioritize work that is:

1. **Helpful** for real site owners
2. **Safe** to ship and maintain
3. **Accessible** to as many people as possible
4. **Honest** about what it can and cannot do
5. **Worth talking about** because it solves real problems well

---

## Community Feedback We Want Most

We especially want feedback on:

- confusing diagnostics or explanations
- missing accessibility improvements
- documentation gaps
- repeated support questions
- bugs that reduce trust or confidence
- safe fixes that would save users real time

---

## Plans vs Promises

This page describes the project’s **direction**, not a binding release contract.

We would rather be honest and flexible than promise more than we can responsibly ship.
