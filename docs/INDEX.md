## Documentation Principles

WP Shadow docs should do three things well:

- tell the truth about current scope
- teach in plain English
- help users act safely

If a document drifts away from the current plugin behavior, the document should be corrected.

## Scope Rules

For public release copy, use the current beta scope only.

That means:

- count only shipped diagnostics and treatments
- distinguish automated treatments from guidance-only treatments
- describe the plugin as local-first
- avoid implying paid or cloud features are part of the current core release
