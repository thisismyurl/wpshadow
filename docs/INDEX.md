## Welcome

WPShadow exists to be a **helpful neighbor** for WordPress site owners.

That means our documentation should do three things well:

1. **Tell the truth clearly**
2. **Teach in plain English**
3. **Help people act safely and confidently**

If you are new here, start with the docs below.

## Current Product Reality

As of **April 3, 2026**:

- **WPShadow Core** is the only currently available product.
- The plugin is **free to use locally**.
- The live plugin UI currently exposes **229 shipped diagnostics** across **11 categories**.
- Public counts should follow **`docs/FEATURES.md`** and the live **`Settings → Diagnostics`** inventory.

### Count Definitions

| Term | Meaning |
|------|---------|
| **Shipped** | Visible in the current release and usable in the live plugin UI |
| **Production-ready** | Shipped and intended for normal site-owner use |
| **Experimental** | Under active development or validation, not yet part of the stable public inventory |
| **Placeholder** | A stub, scaffold, or incomplete implementation |

## Philosophy First

Everything in WPShadow should reflect the core philosophy:

- **Helpful Neighbor** — advice, not pressure
- **Free as Possible** — local value stays free
- **Accessibility First** — nobody left behind
- **Safe by Default** — explain risk, provide confidence
- **Privacy First** — consent before collection
- **Talk-About-Worthy** — quality good enough to recommend naturally

If a document, feature, or workflow conflicts with those principles, the philosophy wins.

See: [`CORE_PHILOSOPHY.md`](CORE_PHILOSOPHY.md)
