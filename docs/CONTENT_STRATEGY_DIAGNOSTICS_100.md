## Category 1: Publishing Frequency & Consistency (10 tests)

### 1.1 Inconsistent Publishing Schedule
**Slug:** `content-inconsistent-publishing`  
**Check:** Analyze post publication patterns over last 90 days. Flag if standard deviation is > 7 days.  
**Why it matters:** Inconsistent publishing hurts reader expectations and SEO momentum.  
**Fix advice:** "You published 3 times in January but only once in February. Readers expect consistency."  
**KB Link:** Understanding Publishing Consistency

## Technical Implementation Notes

### Data Sources
- **WordPress Core:** `wp_posts`, post meta, taxonomies
- **Google Analytics:** Time on page, bounce rate, traffic sources
- **Google Search Console API:** Queries, impressions, positions
- **Readability APIs:** Flesch-Kincaid, Gunning Fog, SMOG index
- **Custom Analysis:** Link checking, image scanning, pattern detection

### Performance Considerations
- Cache readability scores (expensive to calculate)
- Batch process link checking
- Use cron jobs for comprehensive audits
- Provide incremental scans for large sites

### Architecture Pattern
Each diagnostic follows standard WPShadow pattern:
```php
class Diagnostic_Content_Thin_Posts extends Diagnostic_Base {
    protected static $slug = 'content-thin-posts';
    protected static $family = 'content-strategy';
    
    public static function check() {
        // Analysis logic
        // Return finding array or null
    }
}
```

### Auto-Fix Treatments
Many content diagnostics are informational (not auto-fixable), but some can have treatments:
- Add featured images (from library)
- Fix broken links (suggest replacements)
- Add schema markup (auto-generate)
- Update dates (add last modified)
- Generate excerpts (AI-assisted)
