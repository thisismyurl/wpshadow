================================================================================
                          SECURITY PATTERNS
================================================================================

The following patterns should be used for all new code:

1. SECURE AJAX HANDLER
   ├─ verify_request() for nonce + capability
   ├─ get_post_param() for sanitized input
   ├─ Activity_Logger for audit trail
   ├─ try/catch with Error_Handler
   └─ Generic messages to users

2. SAFE FILE OPERATIONS
   ├─ Security_Validator::validate_file_path()
   ├─ realpath() to resolve symlinks
   ├─ Boundary check (path must be within allowed dir)
   └─ WP_Filesystem API for file access

3. SECRET MANAGEMENT
   ├─ Secret_Manager::store() for encryption
   ├─ Secret_Manager::retrieve() for decryption
   ├─ Secret_Audit_Log for access tracking
   └─ Environment variables for production

4. DATABASE QUERIES
   ├─ $wpdb->prepare() with placeholders
   ├─ No variable concatenation in SQL
   ├─ WordPress APIs first (get_posts, get_comments, etc.)
   └─ Custom tables with $wpdb->prefix

5. EXTERNAL API CALLS
   ├─ wp_remote_get/post with timeout & SSL verify
   ├─ Check response code and size
   ├─ Sanitize HTML before caching
   └─ Error logging for failed requests
