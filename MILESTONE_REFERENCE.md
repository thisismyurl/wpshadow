# Milestone Quick Reference

**Unified across all TIMU Suite plugins** (Core, Image Support, License Support, Vault Support, Media Support, Format Spokes)

| Milestone | Due Date | Purpose | Status |
|-----------|----------|---------|--------|
| **M01** | 2026-01-31 | Foundations (admin UI, catalog, licensing, compliance) | IN PROGRESS |
| **M02** | 2026-02-29 | Core Features (Vault, image processing, license verification) | PLANNED |
| **M03** | 2026-03-31 | Diagnostics & Bootstrap (health checks, safe mode, configs) | PLANNED |
| **M04** | 2026-04-15 | Multisite & UX (network policies, discovery, privacy UI) | PLANNED |
| **M05** | 2026-04-30 | Templates & Dependencies (versioning, WP Plugin Dependencies, signatures) | PLANNED |
| **M06** | 2026-04-30 | Privacy & Compliance (GDPR/CCPA, export/erase, audit logging) | PLANNED |
| **M99** | Ongoing | Vision & Future (backlog, community requests, nice-to-haves) | BACKLOG |

## Rules

✅ **Every issue must have a milestone** (M01–M06 or M99)
✅ **All repos use the same milestone structure** (synchronized)
✅ **No scope creep within 2 weeks of milestone due date**
✅ **Move lower-priority items to M99** if a milestone is at risk
✅ **Close completed milestone** and publish release notes monthly

## For Contributors

1. **Creating an issue?** Assign it to the appropriate milestone
2. **What milestone should I use?** See [MILESTONES.md](MILESTONES.md) for detailed scope per plugin
3. **Issue doesn't fit?** Put it in M99; it will be promoted during planning cycles
4. **Have a question?** Open an issue labeled `planning` + `roadmap`

See [MILESTONES.md](MILESTONES.md) for full documentation and repository-specific scope.
