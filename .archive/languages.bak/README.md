# WPShadow Language Files

This directory contains translation files for the WPShadow plugin.

## Files

- **wpshadow.pot** - Translation template (POT file)
  - Contains all translatable strings from the plugin
  - Use as a base for creating new translations
  - Last updated: Check file timestamp

## Creating Translations

### For Translators

1. Download the `wpshadow.pot` file
2. Use a translation tool (like Poedit) to create a new translation:
   - Open `wpshadow.pot` in Poedit
   - Click "Create new translation"
   - Select your language and country
   - Translate all strings
   - Save as `wpshadow-{locale}.po` (e.g., `wpshadow-de_DE.po`)
3. Poedit will automatically create the `.mo` (compiled) file
4. Upload both files to the `languages/` directory
5. WordPress will automatically use your translation

### For Developers

To regenerate the `.pot` file with updated strings:

```bash
# Install WP-CLI if not already installed
# Then run:
wp i18n make-pot . languages/wpshadow.pot --skip-js --domain=wpshadow
```

## Supported Locales

Add new language translations in the following format:

- `wpshadow-{language_code}.po` - Human-readable translation file
- `wpshadow-{language_code}.mo` - Compiled translation file (auto-generated by Poedit)

Examples:
- `wpshadow-de_DE.po` - German (Germany)
- `wpshadow-fr_FR.po` - French (France)
- `wpshadow-es_ES.po` - Spanish (Spain)
- `wpshadow-it_IT.po` - Italian
- `wpshadow-ja.po` - Japanese
- `wpshadow-zh_CN.po` - Chinese (Simplified)

## Text Domain

All strings in WPShadow use the text domain: `wpshadow`

Example usage in code:
```php
__( 'String to translate', 'wpshadow' )
esc_html__( 'Escaped string to translate', 'wpshadow' )
_e( 'String to echo', 'wpshadow' )
```

## Translation Tools

- **Poedit** (recommended) - https://poedit.net/
- **GlotPress** - WordPress translation platform
- **Weblate** - Hosted translation management
- **Loco Translate** - WordPress plugin for browser-based translation

## Language Pack Support

When translations are complete and validated, they can be:
1. Submitted to the official WordPress translation team
2. Distributed directly through the plugin repository
3. Used locally on your WordPress installation

Place `.po` and `.mo` files in this directory, and WordPress will automatically detect and apply them based on your site's language setting.
